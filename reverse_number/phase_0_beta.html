Old approach

Before introducing the "old approach", let's first look at the number from the mathematic lens.
12345 = 10000 + 2000 + 300 + 40 + 5
This can be intepreted more under the combination of 10 multiplers
12345 = 1*10^4 + 2*10^3 + 3*10^2 + 4*10^1 + 5*10^0
From the idea observed from the len of mathematician, we see that every unsigned integer can always be represented under the combination of 10 multiplers. Now, let's look at what we have to do. To reverse number, for example, 12345, it's must become 54321, which mean that
12345 = 1*10^4 + 2*10^3 + 3*10^2 + 4*10^1 + 5*10^0
Now, become
54321 = 5*10^4 + 4*10^3 + 3*10^2 + 2*10^1 + 1*10^0
At first, it's 1*10^4, later, it become 5*10^4, and so on.

Column 4 3 2 1 0
Number 1 2 3 4 5

As eye can discern, each individual digit is multiplied with 10 to the power of column number. With little help with divide(/) and modulus(%) we can get the digit then multiply it with corresponding column.

We will use small number here so a demostration, let's say 123. Now, we need to get the last number(it's number 3). Using modulus, we get
3 = 123 % 10
OK, that good, we now need to get number 2. Let's use modulus again, this time % 100 will give us 2. Wait a minute, it's really 2. Let's try it, shall we?
123 % 100 = 23(that's not 2)
At this moment, our hypothesis seems to be wrong, isn't it?
123 = 100 + 23.
Therefore, % 100 will result in 23. But how about 120. 120 % 100 = 20.
Yup, it doesn't work, either. But wait, we got 20, 20 / 10 = 2. This is the moment of truth.
(123 - 3) = 120
120 % 100 = 20
20 / 10 = 2
 It seems like a long step to deal with when it come to number 1.
((123 - 20 - 3) % 1000) / 10 = 1 Yup, pretty long way, and we have to deal with 1000, what if the number is 1234567890987, how many power of 10 do we have to deal with. I think now we should use the power of computer. Let's say I have a power function named 'floor'. floor(12.5) = 12, floor(4.0) = 4. This function return the integer part of any real number. Now, our method will be reduced to just a few lines with small number. floor(123 / 10) % 10 = floor(12.3) % 10 = 12 % 10 = 2
That's cool, but what about, number 1. Well, let's said it now, we change our mind, we don't need to reverse number 123, we just need
12 (=floor(123 / 10)). Getting 1 is just 1=floor(12 / 10) % 10. Simple, right?

Up until now, we already figure out how to get each digit individually. Let's summarize it:
3 = 123 % 10 2 = floor(123 / 10) % 10 1 = floor(floor(123 / 10) / 10) % 10
Pretty simple, right? Now, in order to get 321, which mean 3*100 + 2*10+1 You may now recognize some similarities between the number of zeroes behind 1 equal to number of missing 'floor'. Well, it's little vague, allow me to elaborate it for you. Number 1 has 2 'floor' appears according to the formula above, and that two 'floor' is the biggest number of floor of the three numbers(1,2,3). Therefore, we mark 1 as standard, number 3 has 0 'floor', it's missing 2 floors, which means it need 2 zeroes(100); on the other hand, 2 has 1 'floor', missing 1 so it has 1 zero(10) and last but not least, 1 has 2 'floor', missing 0 so it has 0 zero(1).

Code

I hope I have explained enough about the workhorse of `reverse number` in mathematics concept. Now it's time for some codes. 


var sum = 0;
var digit = 0;
var num = 321;
while (num != 0) {
	digit = num % 10;
	sum = sum * 10 + digit;
	num = Math.floor(num / 10);
}

The code is self-explanatory itself. However, it's a little bit different from the mathematics way. Let me crack the code into a much familiar way. First, we have some declaration of variable. A digit and sum, and default value of num = 321. Then we go to the while loop. The condition here to execute the while-loop body is the num must be different from 0. At start, we assume num is 321; therefore, obviously, 'num != 0' must be true. Then we get the number 1 by 'num % 10', after that, we take 'sum * 10 + digit', which is '0*10+1' = 1 => sum now is 1. Last but not least, we divide num by 10, which results in 32.1 and flooring it to get 32 => num is now equals 32. After the whole while-loop body is executed, we go back to check the condition 'num != 0', which turns out to be true, and we repeat the whole process again with value of 'num = 32'.
